About PlanetHTML

PlanetHTML combines source formatting, realistic project preview and visual DOM repair in one browser workspace.

Source tools

Beautify or minify HTML, edit optional CSS, copy the result, download the updated entry page and undo or redo structural visual changes.

Project preview

Open one HTML file or select a complete folder. PlanetHTML detects entry pages and resolves referenced local CSS, JavaScript, images, fonts and media to temporary browser URLs. External Bootstrap, Tailwind browser-CDN and other web resources remain linked normally.

Visual editing

The preview runs in a sandbox with a small communication bridge. In Visual edit mode, click an element to edit its ID, classes or text. Drag it onto another element and choose its position naturally by dropping near the top, center or bottom. PlanetHTML repeats that operation against the clean source DOM, so temporary preview URLs and tooling code never leak into the result.

Important limitations

  • Build-time frameworks still require compiled browser assets; this tool does not run npm, Vite, Webpack or a Tailwind build step.
  • Server-side templates and backend routes cannot execute in a static browser preview.
  • JavaScript modules with complex relative dynamic imports may need their normal development server.
  • Folder permissions last only for the current page session; source and extra CSS autosave locally, but browsers do not persist uploaded folder handles.

Privacy

Project files stay in browser memory and are never uploaded.
